Lesson 3: Adding CSS Style to HTML Structure

Repetition

Repetition is strongly discouraged for the following reasons:

Programming Errors: The more lines of code that is written, the more likely an error will be made by the programmar. Syntax errors are one of the main types of errors and this can be minimized by the use of CSS which will be discussed below.

Visual Consistency: When creating a webpage, you want it to look appealing and easy to follow. To do this, CSS allows class styles to be formed for boxes that have the same style throughout the webpage. This allows headers, body text, titles and so forth to have a consistent look and feel.

Webpage Bandwidth: Webpage bandwidth and size depends on the number of lines of HTML code among other things such as image size. To reduce the amount of lines of code, CSS allows the programmer to write a class style and refer to that repeatedly instead of writing repetitive lines of code.

The Use of CSS

CSS is an acronym for Cascading Style Sheets. It allows programmers to change the style and "feel" of related HTML elements. Similar HTML elements are given the same "class" name. Each class has it's own defined style where the programmar can select the font weight, font style, as well as other items.

Input >>> Function >>> Output

What is a function (Dave calls them "procedures")?

A function takes one or more inputs, mainuplates/processes those inputs, and outputs one or more things.

What is the difference between making and using a function?

To make a function you need to start you code with the word def followed by a function name and parameters. These parameters will be replaced by another value when the function is used. The body of the function includes the instructions for how to process the parameters and what to return.

Example: def function_Name(parameters)

To use the function, you write the name of the function and include your parameter(s) in the parentheses.

Example: def function_Name(Actual Parameters)

How do functions help programmers avoid repetition?

Functions call be called over and over again by the programmer throughtout the program. This reduces the amount of code a programmer needs to write and it minimizes the risk of making syntax errors while writing the program.

What happens if a function doesn't have a return statement?

If a function does not have a return statement, the function is essentially not producing an output that can be used outside of the function itself. If the function is called, the output would be None.

Object Oriented Programming (OOP)

What is it used for?

Object oriented programming allows for the creation of objects which are also known as "Classes". Classes allow a programmer to create a set of functions and define certain variables and store it in a class. This class can then be called as an "Instance" in another program or function and it will have access to all of the functions and variables that was stored in the class. A class also adds a layer of protection from other parts of the program. Since data is defined inside a class, the code inside that class will not be able to accidently access other program data. This allows programmers to create "cleaner" looking code and to avoid repetition. OOP also lets the programmer to maintain and modify the existing code by creating new instances or objects with minor changes to the existing one. In addition, OOP provides a defined structure for programs that have abstract datatypes.

Key OOP Vocabulary

Class - It is considered a "blueprint" for how to perform something.

Object or Instance - Classes are used to create objects or instances. For example: brad = class Name(), brad would be considered the object or instance.

Method - A method can also be considered a function. It provides a set of data or generally performs scripting tasks.

Module - A python file that includes functions and classes.

Library - A library contains a group of programs or modules that perform specific tasks. For example, the Python standard library includes modules such as len(), abs(), and open().

Class Variables - These are variables that are defined within a class. They are usable only within the class. Outside the class, the variable is considered undefined.

Inheritance - Inheritance allows one class to use another class' data or properties. An example is: brad = class Name(Age). The class name "inherits" the class properties from class Age.

Method Overriding - When a class inherits properties/data/functions from another class it can be overridden by method overriding. If a function is defined in both class', the class that is not inherited has priority when the class is called.

Abstraction - OOP allows a programmer to extract items from something in order to keep the most relevant data. The purpose of this is to keep things simple and efficient.
